## 3. Deposit happy path

- [ ] In `#deposit`, run `/deposit`. The deposit modal opens
  pre-blank.
- [ ] Submit with: char=<test-char>, realm=Stormrage, region=EU,
  faction=Horde, amount=5000. Bot replies `Ticket opened: <link>`
  ephemeral.
- [ ] A new private channel `deposit-N` is created under
  `Banking`; the open embed (HOUSE blue, "Submitted") is posted.
- [ ] `#cashier-alerts` receives a fresh post tagging `@cashier`
  with the new ticket.
- [ ] In the `deposit-N` channel, run `/claim`. Bot replies
  `✅ Claimed`. The claimed embed (GOLD) replaces the open one.
- [ ] Verify the claimed embed carries the anti-phishing line
  ("⚠️ The cashier will NEVER send a trade request first").
- [ ] In-game: trade 5000g from the test user to the cashier
  char. Open `/confirm` modal in `deposit-N`; type `CONFIRM`;
  the confirm modal accepts.
- [ ] Bot replies `Confirmed`. The confirmed embed (WIN green) is
  posted with the new balance shown.
- [ ] In `#audit-log`, the corresponding ticket-claim and
  ticket-confirm events are visible with the right colours.
- [ ] Run `/balance`; bot reports `5000g`.

## 4. Withdraw happy path

- [ ] In `#withdraw`, run `/withdraw` and submit with amount=2500.
  The open embed shows `amount: 2,500g`, `fee: 50g (2%)`,
  `amount_delivered: 2,450g`.
- [ ] In the withdraw ticket channel, run `/claim` (same cashier).
- [ ] In-game: trade 2450g from the cashier char to the user.
- [ ] Run `/confirm`, type `CONFIRM`. Bot replies `Confirmed`.
- [ ] `/balance` now shows `2500g` (5000 − 2500 = 2500).
- [ ] Treasury check: run `/admin-treasury-balance`. Shows `50g`
  (the captured fee).

## 5. Cancel paths

- [ ] Open another `/deposit` (any amount). Before any cashier
  claims, run `/cancel-mine` in the ticket channel. Bot replies

- [ ] Open a `/withdraw` (e.g. 500g). Before claim, run
  `/cancel-mine`. Embed switches to BUST red "REFUNDED" (the
  refund text — withdraw cancels restore the locked balance).
- [ ] `/balance` confirms the locked amount returned.
